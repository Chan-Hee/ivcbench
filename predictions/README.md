# Prediction bundles — the deposited model-output layer

Each `.npz` here is one **(cluster × model × split)** evaluation, holding the model's predictions plus everything
the frozen metric code needs to recompute the scores **without raw data, checkpoints, or a GPU**. This is the layer
that makes the *evaluation itself* independently checkable (predictions → metrics → tables → figures), the standard
set by scPerturBench, PertEval-scFM, NCBench, and scArchon.

## Format (`ivcbench` prediction bundle)

| key | shape | meaning |
|---|---|---|
| `pred_cells` | float32 [n_test, n_genes] | model-predicted per-cell expression on the test fold |
| `test_cells` | float32 [n_test, n_genes] | observed (ground-truth) per-cell expression |
| `cell_strata` | object [n_test] | stratum label per test cell (perturbation / lineage / donor …) |
| `control_mean` | float32 [n_genes] | matched control mean (the Δ baseline) |
| `genes` | object [n_genes] | gene (HVG) names |
| `exclude_gene_idx` | int [k] *(opt.)* | leak-safe metric exclusions (on-target gene / training-fold response panel) |
| `pca_components`, `pca_mean` | float32 | train-cloud PCA-50 basis, so energy distance reproduces **exactly** |
| `cluster`, `model`, `split` | scalar | provenance |

A compact alternative (per-stratum `pred_means`/`obs_means` instead of per-cell arrays) reproduces Pearson-Δ only;
the per-cell form above reproduces **both** axes.

## Reproduce the scores

```bash
make reproduce-eval                                    # scores every bundle under predictions/
python ../scripts/reproduce_eval.py 'example/*.npz'    # or a glob
```

The reported Pearson-Δ and energy distance match `results/<cluster>/results_raw.csv` to 1e-6 for the predictive
baselines (verified in `tests/test_reproduce_eval.py`).

## What is here

- **`example/`** — a small *synthetic* bundle set (OP3-shaped fixture, the `make pilot` data) so `make reproduce-eval`
  runs out-of-the-box and demonstrates the format. These are **not** the paper's numbers.
- The **real per-model bundles** for C1–C5 are regenerated by re-running the models with `IVCBENCH_PRED_DUMP=<dir>`
  set (the runner and every model script write one bundle per evaluation as a by-product of any run) and are archived on
  Zenodo alongside the result tables.

## Regenerating the bundles (re-running the models)

Set `IVCBENCH_PRED_DUMP=<dir>` and run; every scored evaluation drops its bundle there.

```bash
export IVCBENCH_PRED_DUMP=predictions/C1
make cluster C=C1                          # core-runner models (baselines, scGen, CPA, GEARS, STATE/CINEMA-OT on Frangieh, ...)
python scripts/state_kang.py --gpu 0       # the bespoke heavy-model scripts also honour the variable
python scripts/scpram_soskic.py --gpu 0    # ... etc.
```

## Reproduction coverage (what reproduces, and how)

| Models | Reproduced by | Both axes? |
|---|---|---|
| All core-runner models (every baseline + scGen, CPA, GEARS, STATE/CINEMA-OT-on-Frangieh, …) | generic `reproduce_eval` | Pearson-Δ **and** energy distance, exact to float32 |
| scPRAM (Kang / Soskic / Frangieh), STATE (Kang / Soskic), PertAdapt (Soskic), chemCPA (OP3) | generic `reproduce_eval` (their scripts dump the exact scoring inputs) | Pearson-Δ **and** energy distance, exact to float32 |
| **CellOT** (C4 Frangieh) | the **deposited** `cellot_frangieh.py` scorer | model-specific — see note |

**CellOT note (honest scope).** CellOT is an optimal-transport model whose prediction is a single *perturbation-agnostic*
pushed-control cloud, and whose deposited score is a **3-seed mean** of a model-specific *asymmetric* energy distance
(the one pushed cloud compared against each observed perturbation cloud, `edist_clouds`). That asymmetric, seed-averaged
scheme is not expressible in the symmetric per-stratum bundle here, so CellOT is **not** scored through the generic
wrapper; its prediction and its scorer (`stratum_align` + `edist_clouds`) are both deposited, and re-running the script
reproduces its numbers. `scpram_validate.py` is a published-anchor R² gate, not a deposited result, so there is nothing
to reproduce there.

**Seed caveat.** A bundle captures one *per-seed* evaluation. For a model whose deposited number is a mean over several
seeds, run all the deposited seeds (the dump fires once per seed; the core single-seed headline rows reproduce exactly).
